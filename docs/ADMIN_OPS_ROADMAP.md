# Ops-консоль: дорожная карта спринтов после спринта 4

**Связь:** [ADMIN_OPS_CONSOLE.md](./ADMIN_OPS_CONSOLE.md) (baseline §4.1–§4.2, §6–§8, §11).  
**Уже зафиксировано чеклистами:** [спринт 2](./ADMIN_OPS_SPRINT2_CHECKLIST.md), [спринт 3](./ADMIN_OPS_SPRINT3_CHECKLIST.md), [спринт 4](./ADMIN_OPS_SPRINT4_CHECKLIST.md).

Этот файл задаёт **порядок и смысл** следующих спринтов. Детальные задачи (PR, пункты `[x]`) — в привязанных чеклистах; где решения ещё зависят от ТЗ или схемы БД, в чеклисте явно стоит блок **«уточнить при старте спринта»**.

---

## Принцип

1. Перед началом спринта: открыть чеклист спринта, пройти блок «уточнить при старте», при необходимости добавить/разбить пункты и только потом мержить код.
2. DoD предыдущих спринтов (ручная приёмка, CI) можно закрывать **в хвосте спринта 5** (Epic N), не смешивая с крупными фичами в одном PR без необходимости.
