## Порядок PR (рекомендуемый)

1. **PR-1** — Epic E (внешние ссылки) + при необходимости малое расширение конфига: не трогает БД и сложные запросы.
2. **PR-2** — Epic D (расширение инфраструктуры / очереди): только Admin API + тесты.
3. **PR-3** — Epic A (список разговоров): контракт + репозиторий + эндпоинт + тесты §9 на отсутствие запрещённых полей.
4. **PR-4** — Epic B (карточка разговора): детальный эндпоинт + очистка `meta` + тесты §9.
5. **PR-5** — Epic C (мутации пайплайна): после стабилизации чтения; отдельный PR из-за риска и аудита.
6. **PR-6** (опционально, отдельный спринт или хвост спринта 2) — Epic F (админ-фронт): зависит от API из PR-3–PR-5.

При узком составе команды допустимо объединить **PR-3 и PR-4** в один PR, если объём остаётся обозримым.
