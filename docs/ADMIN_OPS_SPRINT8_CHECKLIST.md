# Ops-консоль: спринт 8+ — чеклист (пост-MVP §4.2)

**Связь:** [ADMIN_OPS_CONSOLE.md](./ADMIN_OPS_CONSOLE.md) (§4.2: лента событий пайплайна, прогресс чанков ASR «k из N»), [ADMIN_OPS_ROADMAP.md](./ADMIN_OPS_ROADMAP.md).  
**База:** спринт 7 (live-обновление) — может частично пересекаться по срокам с подготовительной фазой ниже.

**Решение по данным (зафиксировано в реализации):** прогресс ASR — целочисленные колонки `transcripts.asr_chunk_total`, `transcripts.asr_chunk_completed` (не текст в `meta`). Лента — таблица **`pipeline_events`**, запись из воркеров; типы событий и ключи `detail` ограничены whitelist-ом в коде.

**Статус реализации (код):** фазы A и B (минимальный набор) и B5 выполнены; C1 и ретенция — backlog. Миграция Alembic: `pipeline_events_sprint8_012`.

## Фаза A: прогресс ASR «k из N»

**Уточнить при старте:** откуда берутся `k` и `N` (колонки БД, `Transcript.meta`, отдельная таблица чанков); согласование с воркерами параллельного ASR.

**Ответ:** колонки БД + атомарный increment для параллельных слайсов (`chain(transcribe_slice, record_asr_chunk_done)`); последовательное чанкование — инкремент после каждого временного окна.

## Фаза B: лента / таблица `pipeline_events` (или эквивалент)

**Уточнить при старте:** состав событий (ASR, diarization, summary, embeddings, ошибки); ретенция; индексы для списка по `conversation_id` и глобальной ленте.

**Ответ:** минимальный набор типов — `asr_started` / `asr_completed` / `asr_failed`, `diarization_started` / `diarization_completed` / `diarization_failed`, `embedding_indexed`. Индексы по `(conversation_id, created_at)`, `created_at`, `(event_type, created_at)`. **Ретенция** и фоновая очистка — не реализованы (backlog).
