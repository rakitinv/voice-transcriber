## 1. Сборочная машина (Windows)

### Подготовка

```text
scripts/release/release.env.example  →  scripts/release/release.env
```

Минимум в `release.env`:

| Переменная | Зачем |
|------------|--------|
| `VT_RELEASE_TAG` | Тег дистрибутива (как в `VERSION` / git tag, напр. `v0.2.0`) |
| `VT_RELEASE_DIR` | Куда положить результат (по умолчанию `dist/release`) |
| `VT_PUBLIC_API_URL`, `VT_WEBUI_ORIGIN`, `VT_ADMIN_*` | Prod URL для сборки фронтов |
| `VT_DOCKER_COMPOSE_PROFILES` | Доп. образы: `gpu`, `gpu-unified`, `scale_llm`, `diarization` и т.д. |
| `VT_EXPORT_INFRA_IMAGES=1` | Положить в tar postgres/redis/minio (если площадка без registry) |

Пример для GPU (split):

```env
VT_DOCKER_COMPOSE_PROFILES=gpu
```

Для unified:

```env
VT_GPU_DEPLOY_MODE=unified
VT_DOCKER_COMPOSE_PROFILES=gpu-unified
```

### Сборка

```bat
scripts\release\build-all.bat
```

Скрипт по цепочке:

1. `build-docker` — базовый стек + профили из `VT_DOCKER_COMPOSE_PROFILES`
2. `build-cli` — wheel/sdist
3. `build-extension` — zip расширения
4. `package-release` — упаковка дистрибутива + `docker save` → `docker-images/*.tar`

Результат:

```text
dist/release/voice-transcriber-<VT_RELEASE_TAG>/
  deploy/docker/          # compose, configs, systemd
  docker-images/*.tar     # образы для офлайн-площадки
  artifacts/cli/
  artifacts/browser-extension/
  scripts/release/        # install-or-update.sh и др.
```

Альтернатива без полного цикла: только Docker — `build-docker.bat`, только упаковка — `package-release.ps1` (если образы уже собраны).

**Registry вместо tar:** задать `VT_REGISTRY` / `VT_REGISTRY_TAG`, собрать образы, `push-registry.sh` — на площадке образы тянутся из registry (без `docker load`).

## 2. Удалённая площадка (Linux)

### Первый раз

1. Скопировать каталог `voice-transcriber-<tag>/` на сервер (scp/rsync).
2. Создать серверный env:

```bash
sudo bash scripts/release/configure-server-env.sh /opt/voice-transcriber-<tag>
sudo nano /etc/voice-transcriber/voice-transcriber.env
```

Обязательно: `VT_JWT_SECRET`, `VT_DATABASE_URL`, `VT_REDIS_URL`, `VT_S3_*`, OAuth, `VT_HF_TOKEN`, публичные URL (`VT_PUBLIC_API_URL`, `VT_ADMIN_WEBUI_ORIGIN` и т.д.).

Для GPU на сервере в том же файле:

```env
VT_COMPOSE_PROFILES=gpu
VT_MAIN_WORKER_QUEUES=asr,cleanup
```

или unified:

```env
VT_GPU_DEPLOY_MODE=unified
VT_COMPOSE_PROFILES=gpu-unified
VT_MAIN_WORKER_QUEUES=asr,cleanup
```

3. Установка/обновление:

```bash
sudo bash scripts/release/install-or-update.sh /opt/voice-transcriber-<tag>
```

Важно: **`bash`**, не `sh`.

### Повторное обновление

Тот же сценарий:

1. Собрать новый дистрибутив на сборочной машине.
2. Скопировать на сервер.
3. `sudo bash scripts/release/install-or-update.sh /path/to/voice-transcriber-<new-tag>`

Скрипт:

- `docker load` из `docker-images/*.tar`
- синхронизирует файлы в `VT_INSTALL_ROOT` (по умолчанию `/opt/voice-transcriber`)
- `compose up -d --no-build` (без пересборки на сервере)
- останавливает конкурирующие GPU/CPU воркеры (`apply_gpu_worker_exclusivity`)
- проверяет `/health`

Секреты и профили — в `/etc/voice-transcriber/voice-transcriber.env`; после правок env нужен `docker compose up -d --force-recreate`, не только `restart`.

## Частые подводные камни

| Проблема | Решение |
|----------|---------|
| `$'\r': команда не найдена` на Linux | CRLF в `release.env` / скриптах: `sed -i 's/\r$//' scripts/release/*.sh scripts/release/release.env` |
| `VT_DOCKER_EXTRA_SERVICES` с пробелами | Только в кавычках в `release.env` |
| На сервере не нужны переменные сборки | Достаточно `VT_INSTALL_ROOT`, `VT_SERVER_ENV_FILE`, `VT_COMPOSE_PROFILES` |
| Фронты с localhost | Пересобрать с правильными `VITE_*` / `VT_PUBLIC_API_URL` в `release.env` **до** `build-all` |
| OAuth админки | `VT_ADMIN_WEBUI_ORIGIN` в runtime env **api**, не только при сборке |

---

**Кратко:** `release.env` → `build-all.bat` → копируете `dist/release/voice-transcriber-<tag>/` на сервер → `install-or-update.sh` (после настройки `/etc/voice-transcriber/voice-transcriber.env`).
