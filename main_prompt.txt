System requirements:

The server must support:

1. Realtime transcription via WebSocket
2. Batch transcription via file upload
3. Multi-thread transcription for long audio
4. Speaker diarization (post processing only)
5. LLM-based conversation summary
6. Search through transcripts
7. OAuth2 authentication (Google and Yandex)
8. Per-user encrypted storage

Audio format:

WebM + Opus

Chunk size configurable:
500ms – 2s

Realtime modes:

Mode 1:
chunk -> ASR -> partial transcript

Mode 2:
window buffer -> ASR -> transcript

Configurable via server settings.

Storage:

Audio and transcripts stored in S3.
